# Implementation Plan: Early Wakeup Log

**Branch**: `003-early-wakeup-log` | **Date**: 2026-05-04 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/003-early-wakeup-log/spec.md`

## Summary

On every app foreground event, query Apple Health for sleep sessions since the last checkpoint. Group raw sleep samples (ASLEEP/CORE/DEEP/REM) into logical overnight sessions. For each session ending before 7am with ≥4 hours of sleep and before 10am, insert a log entry with value `1` against the hard-coded Early Wakeup metric. Deduplication is free via the sleep sample UUID as log entry ID. Manual resync in Settings clears and rebuilds the current period.

## Phase 0: Research Summary

See [research.md](research.md) for full rationale. Key decisions:

| Unknown | Decision |
|---------|----------|
| Sleep data API | `getSleepSamples` — already in installed library; returns individual phase samples |
| `value` field type | Native returns string (ASLEEP/CORE/DEEP/REM/INBED/AWAKE), not number — must cast |
| Session grouping | Cluster asleep-type samples with ≤2h consecutive gap; wake = max end; duration = sum |
| Asleep-type phases | ASLEEP, CORE, DEEP, REM — count toward duration and set wake time |
| Deduplication | UUID of last-ending asleep sample in session as log entry ID |
| Permission | Add `SleepAnalysis` to existing `initHealthKit` call alongside `Workout` |
| Checkpoint | MMKV `'sleep:last_sync_at'` in existing `createKV('health-sync')` storage |
| Resync dispatch | `resyncAppleHealthMetrics` dispatches by metric ID to correct query function |

## Phase 1: Design

### `requestHealthPermission()` change

Add `AppleHealthKit.Constants.Permissions.SleepAnalysis` to the read permissions array:

```ts
permissions: {
  read: [
    AppleHealthKit.Constants.Permissions.Workout,
    AppleHealthKit.Constants.Permissions.SleepAnalysis,
  ],
  write: [],
}
```

### New internal constant

```ts
const SLEEP_LAST_SYNC_KEY = 'sleep:last_sync_at';
const EARLY_WAKEUP_METRIC_ID = '1b0558fb-9594-41db-bb2e-bb0f0621b8fc';
```

### New internal type

```ts
interface SleepSession {
  id: string;         // UUID of last-ending asleep sample (dedup key)
  wakeTime: Date;     // max(endDate) of asleep-type samples
  sessionStart: Date; // min(startDate) of asleep-type samples
  durationMinutes: number; // sum(duration) of asleep-type samples
}
```

### `querySleepRawSamples(since: Date): Promise<RawSleepSample[]>`

- Calls `AppleHealthKit.getSleepSamples({ startDate, endDate: now, ascending: true })`
- Casts `value` to string (native/TS type mismatch)
- Filters to asleep-type values: `ASLEEP`, `CORE`, `DEEP`, `REM`
- Returns sorted ascending by startDate

### `groupIntoSessions(samples: RawSleepSample[]): SleepSession[]`

Pure function, no I/O:
- Iterates sorted samples
- Starts a new session when `sample.startDate - previousSample.endDate > 2 hours`
- For each session: `id` = UUID of sample with max endDate; `wakeTime` = max endDate; `sessionStart` = min startDate; `durationMinutes` = sum of durations

### `syncSleepSessions(): Promise<void>`

- Guard: `if (isLocalMode) return`
- `requestHealthPermission()` → return silently on `false`
- Read `SLEEP_LAST_SYNC_KEY` from MMKV (default: 90 days ago)
- `logger.info('Sleep sync started', { since })`
- `querySleepRawSamples(since)` → `groupIntoSessions()`
- Filter sessions: `durationMinutes >= 240 AND wakeTime.hour < 10`
- For each qualifying session where `wakeTime.hour < 7`:
  - `insertLogEntry({ id: session.id, metricId: EARLY_WAKEUP_METRIC_ID, value: 1, loggedAt: session.wakeTime, sessionStartAt: session.sessionStart.toISOString() })`
- After loop: advance `SLEEP_LAST_SYNC_KEY` to wakeTime of the last **qualifying** session (not just the last session)
- `logger.info('Sleep sync complete', { sessions, logged, queued })`

### `useHealthSync` change

Add `syncSleepSessions()` call alongside existing `syncRunningWorkouts()`:
- On mount: call both
- On foreground event: call both (parallel via `Promise.all`)

### `resyncAppleHealthMetrics()` change

Fix the current hard-coded `queryRunningWorkouts` call to dispatch by metric ID:

```ts
async function syncMetricForPeriod(metricId: string, since: Date): Promise<SyncEntry[]> {
  if (metricId === RUNNING_METRIC_ID) {
    return (await queryRunningWorkouts(since)).map(w => ({
      id: w.id, value: w.distanceKm,
      loggedAt: w.endDate, sessionStartAt: w.startDate.toISOString()
    }));
  }
  if (metricId === EARLY_WAKEUP_METRIC_ID) {
    const sessions = groupIntoSessions(await querySleepRawSamples(since))
      .filter(s => s.durationMinutes >= 240 && s.wakeTime.getHours() < 10 && s.wakeTime.getHours() < 7);
    return sessions.map(s => ({
      id: s.id, value: 1,
      loggedAt: s.wakeTime, sessionStartAt: s.sessionStart.toISOString()
    }));
  }
  logger.info('Unknown apple_health metric during resync — skipping', { metricId });
  return [];
}
```
